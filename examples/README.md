# Examples — real output from the Digital Journalist

Two finished, fully-cited articles, both generated by the system. They share **one** research dossier —
the second was spun off the first with `dj-write` (`outFolder` ≠ `srcFolder`), with **no re-research** —
which is why they're mutually consistent.

| Folder | Article | Words | Sources |
|--------|---------|------:|--------:|
| `the-ai-cost-reckoning/` | *The AI Cost Reckoning: Does Replacing Workers With AI Actually Save Money?* | 3,682 | 28 |
| `the-ai-job-apocalypse/` | *The AI Job Apocalypse: Is the Fear Real?* | 3,552 | 25 |

Each folder shows the full machinery:

- `article.md` — the finished piece (NYT/WSJ/Bloomberg standard: anecdotal lede → nut graf → evidence
  sections → steelmanned contrarian turn → "What we don't know" → kicker).
- `master.md` — the consolidated evidence dossier (quote bank, fact/data ledger, source index).
- `research/` — per-lens findings, `verification.md` (the adversarial fact-check), and `synthesis.md`.
- `titles.md` — 10 SEO-optimized headline options. `sources.md` — tiered source list. `meta.json` — run metadata.

> These are static showcases. Internal path references (e.g. a `srcFolder` in `meta.json`) point at the
> original `articles/<slug>/` location, not `examples/` — they're for reading, not re-running.

Want to see how it's made? Read [the Architecture wiki](../wiki/Architecture.md), then run the journalist
on your own topic.
